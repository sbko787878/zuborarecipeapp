# Zubora Recipe App

「献立を考えるのが面倒」という悩みを解決する、スワイプ操作で直感的にレシピを選べるモバイル特化型Webアプリケーションです。

職業訓練校での半年間にわたる学習（Java/SQL/Spring Boot等）の集大成として、チームで開発したWebアプリケーションです。
私はバックエンドエンジニアとして、DB設計からサーバーサイドのロジック実装、Spring Securityによる認証機能の導入などを担当しました。
また、Gitを用いたソースコード管理や、チームメンバーとの仕様調整にも注力しました。

## 🛠 ローカルでの実行方法（Docker）
Dockerfileを使用して、環境を汚さずに実行可能です。

1. **イメージのビルド**
   ```bash
   docker build -t zubora-app .
   ```
2. **コンテナの起動**
```bash
  docker run -p 8080:8080 \
   -e DB_URL=jdbc:mysql://host.docker.internal:3306/zubora_recipes \
   -e DB_USERNAME=root \
   -e DB_PASSWORD=your_password \
   zubora-app
  ```

## ✨ こだわったポイント

1. **思考コストを最小化した、ストレスフリーな検索UX**
: 「献立を考えるのが面倒」というユーザー（ずぼらな人）の心理に寄り添い、文字入力の手間を一切排除し、直感的な**アイコンタップのみで完結する絞り込み検索**を実装しています。

2. **検索における「デッドエンド」の完全排除**
: JavaScriptを用いた動的なカテゴリ制御により、選択可能なレシピが存在しないアイコンをリアルタイムで非表示化。「検索結果0件」という負のUX（デッドエンド）を未然に防いでいます。

3. **インフラの一気通貫な構築と運用**
: AWS（EC2）のプロビジョニングからネットワーク設定、デプロイまでを独力で完遂。単なる開発に留まらず、プロダクトを外部公開し運用するまでの一連のサイクルを経験しました。

## 🛠 デプロイとセキュリティの改善記録

開発の途中で、データベースの接続情報などの機密情報をリポジトリに含めてしまうという失敗をしました。
しかし、このミスを真摯に受け止め、プロの現場でも通用する「Docker」と「環境変数」を組み合わせた安全な運用方法を導入し、システム構成を刷新しました。

### 1. 失敗から学んだ「環境変数の分離」
機密情報をソースコードから完全に切り離し、外部から注入する方式に変更しました。
* **ローカル環境**: `.env` ファイルを使用し、Gitの管理対象から外すことで漏洩を徹底防止。
* **本番環境（AWS）**: Dockerコンテナ起動時にOSの環境変数を経由して認証情報を渡す、セキュアな構成を実現。

### 2. Docker導入による環境の共通化
AWS（EC2）へのデプロイを確実にするため、アプリケーションをDockerイメージ化しました。
* 開発環境と本番環境の差異をなくし、「どこでも動く」状態を構築。
* Dockerのブリッジネットワークについて知り、コンテナ内からホスト側のDBへ安全に通信させる設定（172.17.0.1の利用）で解決しました。

### 📝 この経験を通して
最初は自分のミスに焦りましたが、「これを解決できれば現場で通用するインフラの知識が身につく」と考え、あえて難易度の高いDocker運用に挑戦しました。
トラブルから逃げずに原因を特定し、新しい技術を使い解決まで持っていけたことは、自走力を養う大きな自信になりました。
